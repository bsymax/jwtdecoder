This tool decodes the JWT header and payload (which are Base64url-encoded, not encrypted). Verifying the cryptographic signature requires the secret key — that stays on your server. The data shown here is what anyone with your token can already see.

How to Use This Tool

  1. Paste your JWT token into the input box above. The decoder auto-detects and parses instantly.
  2. View the Header — shows the signing algorithm and token type.
  3. View the Payload — shows all claims (user data, timestamps, permissions) with syntax highlighting.
  4. Check Timestamps — iat, exp, and nbf are converted to human-readable dates with relative time.
  5. Click Copy on any section to copy the decoded JSON to your clipboard.

FAQ

Is it safe to paste my JWT here?

Yes. Everything runs locally in your browser — your token never leaves your device. No server, no tracking, no data collection. But remember: the decoded content is what anyone with your token can already see. Never share your tokens publicly.

What's the difference between decoding and verifying?

Decoding reads the header and payload (Base64url). Verifying checks the cryptographic signature using the secret key — only your server can do that. This tool decodes; it does not verify signatures.

Why does my JWT have three parts?

Every JWT is header.payload.signature. The header says how it's signed, the payload carries the data (claims), and the signature proves it wasn't tampered with.
